## 0. 直接回答：<1.5 的 Sharpe 有实盘部署意义吗？

**没有。而且真实情况比 1.5 这个门槛更差。**

实跑结果（关键事实）：

| 策略 | 年化 | Sharpe | 最大回撤 | 真相 |
|---|---|---|---|---|
| S1 ST困境反转 | 49.3% | **1.47** | -45.3% | ⚠️ **只跑了 1 个月**就被止损打断 → Sharpe 统计无效 |
| S5 壳资源深度价值 | 55.9% | 0.83 | **-51.4%** | 跑满全程的最高收益；Calmar≈1.09；壳价值 alpha 随注册制衰减 |
| S3 极小小市值轮动 | 41.9% | 0.67 | -43.2% | 纯市值 beta，不是 alpha |
| SB 多因子(Size+PB) | 41.6% | 0.67 | -43.2% | 跑满全程最佳复合 |
| SC/SA 低波类 | 34/27% | 0.66/0.61 | -52/-47% | — |
| E–H 学术模板 | ~5–9% | 0.04–0.15 | ~-45% | regime/sentiment/ensemble/mtf 全部≈躺平，勉强跑赢基准 |

**三点诊断：**

1. **1.47 是假象。** 它来自一个只运行 1 个月的回测（`robust_start` 已告警）。**跑满全程的存活策略，Sharpe 真实区间是 0.6–0.85。**
2. **还要再打三次折扣**（当前引擎未建模或建模不足）：
   - **幸存者偏差** — universe 是"今天还活着"的 586 只，退市/长期停牌的微盘股被静默剔除。`grep 退市/delist = 0 命中`，这是微盘回测最致命的偏差。
   - **非线性冲击成本** — 引擎只有固定 0.2% 滑点（`engine.py:170`）；微盘真实冲击 ≈ `(下单额/ADV)^0.5`，集中持仓单边可达 1–3%，月度调仓 20–30 只复利侵蚀。
   - **多重检验** — 跑了 19 个策略只报最好的，需用 Deflated Sharpe（Bailey & López de Prado）按试验次数打折。
   叠加后，**可部署净 Sharpe 极可能 < 0.5。**
3. **真正的约束不是 Sharpe，是回撤。** -45% ~ -65% 的回撤，连最优的 S5 都 -51%。一个能腰斩的集中多头微盘组合，无论 Sharpe 多少，都无法上规模部署——2024-01 微盘股数周 -30% 的踩踏就是悬在头顶的尾部风险。

**部署门槛（建议）：** 真实成本后 OOS Sharpe ≥ 1.0（目标 1.5）、Calmar ≥ 0.5（回撤 < 30%）、跨起点/牛熊稳定、容量可量化（ADV 约束）、Deflated Sharpe > 0。

> 结论：先把"数字做真"（WS-A），再加"真正分散的新 alpha"（WS-B/C），最后套"风险护栏"（WS-D）。当前策略几乎全部押在 **市值 + 深度低 PB**——这恰恰是最拥挤、最像 beta、最怕踩踏的来源。

|---|---|---|---|---|---|---|
| **A1** | **短期反转 / 流动性提供** | 散户过度反应 + 流动性溢价 | 极高（散户主导筹码） | 低（容量受限→持久） | OHLCV✓ + 真实成交额 | mom20d 当成趋势用，反向未挖 |

## 4. 算法实现原理与模式

> 多数项目**不需要新 alpha** 就能提升 Sharpe/Calmar，并直接攻击 -50% 回撤这个真正的约束。

**4.1 可信度 / 现实性模式（P0，先做）**
- **PIT 幸存者无偏 universe**：用上市/退市日做 as-of join，按调仓日重建成份（含已退市），再跑全部策略，报告 Sharpe 差值。（UA1）
- **ADV 平方根冲击 + 容量上限**：`impact_bps = k·sqrt(下单额/ADV)`，ADV 由 `成交额=close·volume` 估；持仓 ≤ ADV 的 c%；结果输出容量(万元)。（UA2）
- **Deflated Sharpe + 试验计数**：按试验次数对最优 Sharpe 打折，杀掉伪赢家。（UA3）
- **起点/regime 稳健性扫描**：滚动起点回测，报告 Sharpe/回撤分布，剔除"只在好起点存活"的策略。（UA4）

**4.2 因子工程模式**
- **标准化管线**：winsorize → 中性化(size+行业) → z/rank → 合成，**对所有因子统一前处理**（算子 group_rank/scale 已具备）。（UC1）
- **去相关合成**：IC 加权，强制两两 |corr|<0.6——这是把当前 Sharpe 0.15 的"伪集成"变成真分散的关键。（UD3）

**4.3 组合构建 / 风控模式（P0-P1，最大回撤收益）**
- **风险护栏 = 波动率目标 + regime 降仓 + 回撤节流**：把 gross 敞口缩放到目标波动；当市场宽度崩塌或微盘指数跌破均线时降仓（崩盘过滤器）；滚动回撤触线再降 gross。与已有的逐日个股止损正交。**预期把回撤从 ~-50% 压向 <-30%。**（UD1）
- **组合构建超越 top-N 等权**：IC 加权 / 风险平价 / 带约束 MVO；单票上限(如 ≤8%)+行业上限；并入 ADV 容量约束。（UD2）

**4.4 验证 / 自适应模式**
- **walk-forward + purged/embargo CV，测试集单次使用**：`research_loop/walk_forward.py` 与 `walk_forward_validation_split` 已具备，需强制纪律：冻结 2025-2026 做唯一最终评估，记录每次访问。（UD4）
- **在线/滚动重拟合因子权重（SGD）**：微盘 alpha 6-12 月衰减，静态权重会过时；`SGDRegressor.partial_fit` + L2，季度重拟合（已在 `ml_factor_mining_research.md` 调研，预期 +0.1-0.3 Sharpe）。（UD5）
